#ПЛАН УЧЕБНОЙ ПРАКТИКИ (УП.01) для обучающихся по 09.02.07 «Информационные системы и программирование» 2 курс, 4 семестр (в рамках освоения профессионального модуля ПМ.01 «Разработка программных модулей») специальности

####1. Вид практики: Учебная практика. 

####2. Способ проведения: Рассредоточенная, на базе научной лаборатории колледжа.

####3. Основание: Рабочая программа профессионального модуля ПМ.01 и учебный план специальности.

####4. Цели практики:

Сформировать первоначальный практический опыт разработки полнофункционального веб-приложения.
Закрепить навыки работы с современными стеком-технологиями: Python (FastAPI), PostgreSQL, Nest.JS/React и Telegram Bot API.
Развить общие и профессиональные навыки в области задач анализа, проектирования, программирования, тестирования и документирования.
####5. Задачи практики:

Проанализировать техническое задание по разработке информационных систем.
Спроектировать архитектуру приложения «клиент-сервер-база данных».
Реализовать backend-API на Python с использованием FastAPI и интеграцией с PostgreSQL.
Разработать клиентское приложение на Nest.JS или React либо Telegram-боте для взаимодействия с API.
Протестировать взаимодействие всех компонентов системы.
Оформить проектную документацию и исходный код в Git-репозитории.
Подготовить отчет и защитить результаты работы.
####6. Место и время пребывания: Место: Учебные информационные технологии колледжа. Срок: 4 недели в соответствии с календарным учебным графиком. Трудоемкость: 144 часа (36 часов в неделю).

####7. Тема итогового проекта: Разработка прототипа информационной системы «Система измерения работ» с использованием технологий Python (FastAPI), PostgreSQL, Nest.JS/React и Telegram Bot.

####8. Содержание работы и календарный график:

#####Неделя 1. Проектирование и подготовка специалистов.

Содержание работы:
Анализ технического задания. Определение требований к системе.
Разработка компонентов схем взаимодействия (API, БД, Frontend, Bot).
Проектирование логической модели базы данных. Создание ER-диаграмм.
Настройка рабочего окружения: установка и настройка IDE VSCode, СУБД PostgreSQL, Python, Node.js.
Создание и настройка Git-репозитория на GitHub. Название начального .gitignore.
Контрольный результат недели:
Утвержденная схема конструкции приложения.
ER-диаграмма базы данных и SQL-скрипт для создания таблиц (init_db.sql).
Созданный и доступный репозиторий на GitHub.

#####Неделя 2. Разработка серверной части (Backend API).

Содержание работы:
Инициализация FastAPI-приложения. Настройка структуры проекта.
Подключение к базе данных PostgreSQL с использованием асинхронных драйверов (asyncpg) и ORM (SQLAlchemy).
Создание Pydantic-моделей для валидации данных и SQLAlchemy-моделей для работы с БД.
Разработка CRUD-эндпоинтов для основных оснований «Задача» (создание, чтение, обновление, удаление).
Добавление условий маршрутизации и обработки ошибок.
Настройка CORS для обеспечения взаимодействия с будущим интерфейсом.
Контрольный результат недели:
Рабочее FastAPI-приложение, запускаемое локально.
Сбор проверенных запросов к API в Postman или Insomnia.
Реализованные основные конечные точки, работающие с тестовой базой данных.

#####Неделя 3. Разработка клиентской части (Frontend/Telegram Bot).

Содержание работы (по выбору студента или по указанию руководителя):
Вариант А (Веб-интерфейс на Nest.JS/React):

Разработка пользовательского интерфейса для отображения задач таблицы и формы создания/редактирования.3. Реализация логики для отправки HTTP-запросов (fetch/axios) к разработанному API.
Изображение отображения данных, полученных с сервера.
Вариант Б (Telegram Бот):
Создание нового бота через @id199142634 (@BotFather) и получение токена.
Разработка бота на айограмме библиотеки (асинхронной).
Реализация команд (например, /start, /tasks, /new) и обработка обратных запросов с кнопок.
Интеграция бота с ранее созданным API: отправка запросов для получения списка задач, создание новой задачи.

Рабочее клиентское приложение (локальный веб-сервер или функционирующий Telegram-бот).

#####Неделя 4. Интеграция, обучение, защита.

Содержание работы:
Комплексная проверка работы всей системы. Сквозное обсуждение.

Написание ключевых модульных тестов для API (с использованием pytest).

Подготовка итоговой презентации (5-7 слайдов) и демонстрационного сценария.

Защита проекта перед аттестационной комиссией.
Контрольный результат недели:
Стабильно работающий прототип информационной системы.
Заполненный Git-репозиторий с качественным кодом и документацией.
Завершенный письменный отчет по ссылке.
