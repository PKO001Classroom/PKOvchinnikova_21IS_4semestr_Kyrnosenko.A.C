## 1. Введение:

- Название системы:GradeMaster - Система управления академической успеваемостью
- Приложение: Интегрированная система выставления, учета и анализа оценок студентов в образовательных учреждениях
- Цель: Полная автоматизация процесса управления академической успеваемостью от выставления оценок до формирования аналитических отчетов

*   Контекст:

- Замена бумажных журналов и ведомостей на цифровую систему
- Интеграция с существующими учебными процессами образовательных учреждений
- Создание единой платформы для всех участников образовательного процесса

## 2. Бизнес-цели:
### 2.1 Стратегические цели:
   
- Повышение эффективности учебного процесса на 30%

- Увеличение прозрачности системы оценивания
- Создание централизованной базы академических данных учреждения

### 2.2 Операционные цели:

- Автоматизация рутинных операций:
- Выставление и расчет итоговых оценок
- Формирование ведомостей и отчетов
- Уведомление участников о изменениях

*   Централизованное хранение данных:

- Единая база академических достижений студентов
- Архив успеваемости за весь период обучения
- Резервное копирование и восстановление данных

*   Доступность информации:

- Круглосуточный доступ к оценкам для студентов

*   Аналитика и отчетность:

- Автоматическое формирование отчетов для деканатов

## 3. Основные требования:
### 3.1 Функциональные требования:
#### 3.1.1 Управление пользователями:
   
*   Регистрация и аутентификация:

- Многоуровневая система регистрации (администратор, преподаватель, студент)
- Подтверждение email при регистрации
- Восстановление пароля через email/SMS:
  - AC1: При нажатии "Забыли пароль?" на странице входа пользователь вводит email, зарегистрированный в системе.
  - AC2: Система отправляет на этот email письмо с уникальной одноразовой ссылкой, действительной в течение 1 часа.
  - AC3: При переходе по ссылке пользователь попадает на защищенную страницу, где может задать новый пароль (минимум 8 символов, буквы и цифры).
  - AC4: После успешной смены пароля система автоматически завершает все активные сессии пользователя и отправляет уведомление на email.
- Двухфакторная аутентификация для администраторов

*   Управление ролями и правами:

- Ролевая модель: Администратор → Преподаватель → Студент → Гость
- Гибкая система разрешений для каждой роли
- Временное предоставление прав доступа
- Аудит действий пользователей

#### 3.1.2 Управление учебным процессом:

*   Создание и управление курсами:

- Создание курсов с детальными описаниями
- Прикрепление учебных материалов и программ
- Настройка структуры курса (модули, темы)
- Управление расписанием занятий и дедлайнами

*   Формирование учебных групп:

- Автоматическое распределение студентов по группам
- Ручная корректировка состава групп
- Импорт списков студентов из CSV/Excel
- Синхронизация с системой "1С:Университет"

#### 3.1.3 Система оценивания:

*   Типы оценок и критерии:

- Оценочная шкала: 5-балльная, 100-балльная, ECTS
- Весовые коэффициенты для разных типов работ
- Рубрики для объективного оценивания
- Система накопительных баллов (балльно-рейтинговая)

*   Процесс выставления оценок:

- Пакетное выставление оценок
- Автоматический расчет среднего балла
- Корректировка оценок с указанием причины
- Система апелляций и пересдач:
  - Студент может подать запрос на апелляцию в течение 72 часов после публикации оценки.
  - Запрос поступает преподавателю, который может: отклонить с комментарием, принять и исправить оценку, назначить повторную защиту (создать новое задание "Пересдача").
  - Все действия фиксируются в аудит-логе.
- Управление файлами и приложениями.
- Система уведомлений (In-App & Email).
- Календарь/Расписание.

#### 3.1.4 Аналитика и отчетность:

*   Индивидуальная статистика:

- Динамика успеваемости по семестрам
- Сравнение с группой/потоком
- Рейтинговая позиция студента
- Прогноз итоговой успеваемости

*   Групповая аналитика:

- Статистика успеваемости по группам и курсам
- Анализ эффективности преподавания
- Выявление "групп риска"
- Формирование рекомендаций

*   Отчеты и экспорт:

- Автоматическое формирование ведомостей
- Экспорт в форматы: PDF, Excel, Word
- Генерация транскриптов и справок
- Интеграция с системой дипломов

### 3.2 Нефункциональные требования:
#### 3.2.1 Производительность:

*  Нагрузочные характеристики:

- Поддержка до 1000 одновременных пользователей:
  - Предложение: "Система должна выдерживать пиковую нагрузку в 1000 виртуальных пользователей (VU) в течение 30 минут в сценарии, имитирующем типичную активность (60% студентов просматривают оценки, 30% загружают работы, 10% преподавателей проверяют задания)".
- Время отклика системы: < 2 секунд для 95% запросов
- Обработка 100+ транзакций в секунду
- Поддержка до 50,000 пользователей в базе
- Загрузка главной страницы авторизованного пользователя: < 1.5 сек.
- Загрузка журнала оценок с 50 студентами: < 3 сек.
- Загрузка интерфейса проверки работы: < 2 сек.

*   Масштабируемость:

- Горизонтальное масштабирование серверной части
- Кэширование часто запрашиваемых данных
- Оптимизация запросов к базе данных
- Использование CDN для статических ресурсов

#### 3.2.2 Безопасность:

*   Защита данных:

- Шифрование передаваемых данных (TLS 1.3)
- Хранение паролей в хешированном виде (bcrypt)
- Регулярное резервное копирование
- Защита от SQL-инъекций и XSS-атак

*   Контроль доступа:

- Ролевая модель авторизации (RBAC)
- Сессии с ограниченным временем жизни
- Защита от brute-force атак
- Мониторинг подозрительной активности

#### 3.2.3 Надежность:

*   Доступность:

- Гарантированная доступность 99.5% (плановые работы - не более 4 часов в месяц)
- Отказоустойчивая архитектура
- Мониторинг работоспособности 24/7
- Автоматическое оповещение о сбоях

*   Целостность данных:

- Транзакционность операций
- Ведение журнала изменений (audit log)
- Проверка целостности данных
- Восстановление после сбоев

#### 3.2.4 Удобство использования:

*   Интерфейс пользователя:

- Адаптивный дизайн (поддержка мобильных устройств)
- Интуитивная навигация
- Минимальное количество кликов для основных операций
- Подсказки и всплывающие подсказки

*   Доступность:

- Поддержка стандарта WCAG 2.1
- Адаптация для людей с ограниченными возможностями
- Альтернативные способы взаимодействия
- Локализация интерфейса
- Эффективность: Опытный пользователь выставляет оценку за работу за 3 шага и не более 30 секунд.
- Ошибки: Количество ошибок, вызванных интерфейсом системы, должно быть менее 5% от всех действий пользователя. Должны быть понятные сообщения об ошибках и пути их исправления.
- Удовлетворенность: Оценивается по стандартизированному опросу SUS после месяца использования.

## 9. Глоссарий (Термины и определения):

- Задание (Assignment): Работа, которую должен выполнить студент (лабораторная, тест, проект).
- Оценка (Grade): Результат оценивания работы по конкретному заданию.
- Итоговая оценка (Final Grade): Расчетная оценка за курс на основе всех оценок с учетом весов.
- Рубрика (Rubric): Таблица с критериями оценивания и описанием уровней достижения.
- Дедлайн (Deadline): Крайний срок сдачи задания. Может быть мягким (принимаются с штрафом) и жестким.
- Поток (Cohort): Группа студентов, обучающихся по одной программе в одном году набора.
- Транскрипт (Transcript): Официальная выписка об успеваемости.

## 10. Сценарии использования (User Story Mapping или Key User Journeys):

- Краткое описание 3-5 ключевых сценариев, которые раскрывают последовательность действий.
- Пример, Сценарий "Сдача курсовой работы":
- Преподаватель создает задание "Курсовая работа", загружает ТЗ, выставляет весовую коэффициенты, дедлайн.
- Студент видит задание в своем курсе, скачивает ТЗ, загружает готовый файл.
- Система уведомляет преподавателя о новой работе на проверке.
- Преподаватель проверяет файл, заполняет рубрику, выставляет баллы, пишет развернутый комментарий.
- Система рассчитывает итоговый балл, сохраняет его в журнал, отправляет уведомление студенту.
- Студент видит оценку и комментарий, при необходимости подает запрос на апелляцию.

## 11. Внешние интерфейсы и интеграции (Interfaces & Integration):

- EI-001: API для отчетов. Описание: REST API для экспорта данных об успеваемости в формате JSON/XML. Частота: По запросу. Ответственность: Система предоставляет данные, внешняя система их запрашивает с аутентификацией.
- EI-002: Импорт пользователей из LDAP/Active Directory. Описание: Ежедневная синхронизация учетных записей, ролей и структурных подразделений. Формат данных: LDAP протокол.
- EI-003: Экспорт данных для "1С:Зарплата и кадры". Описание: Выгрузка информации о стипендиатах по итогам сессии. Формат: XML, совместимый с конфигурацией "Университет".
