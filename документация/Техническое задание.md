## 2. Функциональные требования

### 2.1. Модуль пользователей
1. Регистрация и аутентификация пользователей
2. Ролевая модель: Администратор, Преподаватель, Студент
3. Управление профилями пользователей
4. Восстановление пароля

### 2.2. Модуль курсов и заданий
1. Создание и управление учебными курсами
2. Создание заданий с критериями оценки
3. Установка сроков сдачи работ
4. Прикрепление учебных материалов

### 2.3. Модуль загрузки работ
1. Загрузка студенческих работ в различных форматах
2. Отслеживание статуса сдачи
3. Возможность повторной загрузки (при настройке)

### 2.4. Модуль проверки работ
1. Интерфейс для оценки работ по критериям
2. Выставление баллов и комментариев
3. Расчет итоговой оценки
4. Отправка результатов студентам

### 2.5. Модуль оценок и отчетности
1. Ведомость успеваемости
2. Статистика по курсам и группам
3. Экспорт данных в Excel/PDF
4. Личный кабинет студента с историей оценок

### 2.6. Система уведомлений
1. Автоматические уведомления о событиях
2. Email-рассылка
3. Внутрисистемные оповещения

## 3. Требования к программному обеспечению

### 3.1. Серверная часть
- **Операционная система:** Windows 10/11 
- **Язык программирования:** Python 3.11 или выше
- **Веб-фреймворк:** FastAPI
- **База данных:** PostgreSQL 17
- **ORM библиотека:** SQLAlchemy
- **Контейнеризация:** Docker 
- **Система контроля версий:** Git

### 3.2. Клиентская часть
- **Фреймворк:** React.js 
- **Язык программирования:** JavaScript/TypeScript
- **UI библиотека:** Material-UI 
- **HTTP клиент:** Axios или Fetch API

### 3.3. Инструменты разработки
- **Редактор кода:** VS Code 
- **Тестирование API:** Postman или Insomnia
- **Проектирование БД:** pgAdmin 
- **Диаграммы:** draw.io

---

## 4. Требования к интерфейсу

### 4.1. Пользовательский интерфейс
- Веб-интерфейс на русском языке
- Адаптивный дизайн для различных устройств
- Интуитивно понятная навигация
- Поддержка современных браузеров (Chrome, Firefox, Edge)

### 4.2. API интерфейсы
- REST API для взаимодействия компонентов
- Документация API в формате OpenAPI/Swagger
- JSON формат обмена данными

## 5. Требования к данным

### 5.1. Структура базы данных
Система должна хранить следующие основные данные:
1. Данные пользователей (студенты, преподаватели, администраторы)
2. Информация о курсах и учебных группах
3. Задания с критериями оценки
4. Сданные работы студентов
5. Оценки и комментарии преподавателей
6. Системные настройки

### 5.2. Форматы файлов
- **Документы:** PDF, DOC, DOCX
- **Архивы:** ZIP, RAR
- **Изображения:** JPEG, PNG
- **Размер файла:** до 50 МБ
