Архитектурная документация GradeMaster
1. Введение:
   
Название системы: GradeMaster - Архитектура системы управления академической успеваемостью
Цель документа: Определение архитектурных решений, компонентов и технологий для реализации бизнес-требований системы GradeMaster
Контекст: Документ описывает архитектурные решения, обеспечивающие выполнение всех бизнес-требований с акцентом на масштабируемость, безопасность и производительность

2. Архитектурные цели и ограничения:
   
2.1 Цели архитектуры:
   
Обеспечение производительности: Поддержка до 1000 одновременных пользователей с временем отклика < 2 секунд
Обеспечение безопасности: Соответствие 152-ФЗ о персональных данных, ролевая модель доступа
Обеспечение надежности: Доступность системы 99.5%, автоматическое восстановление после сбоев
Обеспечение масштабируемости: Горизонтальное масштабирование под нагрузку
Обеспечение поддерживаемости: Четкое разделение ответственности между компонентами

2.2 Архитектурные ограничения:
Технические: Веб-интерфейс как единственный способ доступа
Интеграционные: Поддержка PostgreSQL/MySQL, REST API для интеграций
Языковые: Полная поддержка русского и английского языков
Временные: Развертывание MVP за 4 месяца

3. Архитектурный стиль и принципы:
   
3.1 Выбранный стиль: Микросервисная архитектура с элементами событийно-ориентированного дизайна
Обоснование выбора:

3.2 Принципы проектирования:
Принцип единственной ответственности: Каждый сервис отвечает за одну бизнес-функцию

Принцип независимости данных: Каждый сервис управляет своими данными

Принцип слабой связанности: Сервисы общаются через API, а не через базу данных

Принцип устойчивости к отказам: Система продолжает работать при сбоях отдельных компонентов

Принцип наблюдаемости: Все компоненты предоставляют метрики и логи

6. Детальное проектирование компонентов:
5.1 Authentication Service (Сервис аутентификации):
   
Назначение: Управление аутентификацией и авторизацией пользователей
Технологии: Node.js + Express.js + JWT
Бизнес-требования: 3.1.1 (Регистрация и аутентификация), 3.2.2 (Безопасность)

API Endpoints:
  POST /api/v1/auth/register     # Регистрация пользователя
  POST /api/v1/auth/login        # Вход в систему
  POST /api/v1/auth/refresh      # Обновление токена
  POST /api/v1/auth/logout       # Выход из системы
  POST /api/v1/auth/reset-password # Сброс пароля
  POST /api/v1/auth/verify-2fa   # Двухфакторная аутентификация

Особенности:
  - JWT токены с коротким временем жизни (24 часа)
  - Refresh tokens с возможностью отзыва
  - Поддержка OAuth 2.0 для внешних провайдеров
  - Интеграция с LDAP/Active Directory
  - Rate limiting: 5 попыток входа в минуту с одного IP
    
5.2 User Management Service (Сервис управления пользователями):
    
Назначение: Управление профилями пользователей и ролями
Технологии: Python + FastAPI + SQLAlchemy
Бизнес-требования: 3.1.1 (Управление ролями и правами), 3.1.2 (Управление учебным процессом)

API Endpoints:
  GET    /api/v1/users           # Список пользователей
  POST   /api/v1/users           # Создание пользователя
  GET    /api/v1/users/{id}      # Получение пользователя
  PUT    /api/v1/users/{id}      # Обновление пользователя
  DELETE /api/v1/users/{id}      # Удаление пользователя
  GET    /api/v1/users/{id}/roles # Роли пользователя
  POST   /api/v1/users/{id}/roles # Назначение роли

Особенности:
  - Ролевая модель RBAC (Admin, Teacher, Student)
  - Аудит всех действий с пользователями
  - Маскирование персональных данных в логах
  - Пакетные операции для импорта пользователей
5.3 Academic Service (Академический сервис):
    
Назначение: Управление курсами, группами и оценками
Технологии: Python + Django + Django REST Framework
Бизнес-требования: 3.1.2 (Управление учебным процессом), 3.1.3 (Система оценивания)

Модули:
  - Course Module: Управление курсами (создание, обновление, архивация)
  - Group Module: Формирование учебных групп
  - Grade Module: Выставление и расчет оценок
  - Schedule Module: Управление расписанием

API Endpoints (Grade Module):
  POST   /api/v1/grades/batch     # Пакетное выставление оценок
  GET    /api/v1/grades/student/{id} # Оценки студента
  GET    /api/v1/grades/course/{id}  # Оценки по курсу
  POST   /api/v1/grades/calculate   # Расчет итоговых оценок
  POST   /api/v1/grades/export      # Экспорт оценок

Особенности:
  - Поддержка разных шкал оценивания (5-балльная, 100-балльная, ECTS)
  - Весовые коэффициенты для разных типов работ
  - Автоматический расчет среднего балла
  - Система апелляций и пересдач
  - Асинхронная обработка пакетных операций
5.4 Analytics Service (Аналитический сервис):
    
Назначение: Аналитика успеваемости и формирование отчетов
Технологии: Python + Pandas + Matplotlib + ReportLab
Бизнес-требования: 3.1.4 (Аналитика и отчетность)

Функциональность:
  - Статистика успеваемости по студентам, группам, курсам
  - Формирование ведомостей и транскриптов
  - Аналитические графики и диаграммы
  - Прогнозирование успеваемости (ML модели)
  - Сравнительный анализ с предыдущими периодами

API Endpoints:
  GET    /api/v1/analytics/student/{id}    # Статистика студента
  GET    /api/v1/analytics/course/{id}     # Статистика курса
  GET    /api/v1/analytics/faculty/{id}    # Статистика факультета
  POST   /api/v1/reports/generate          # Генерация отчета
  GET    /api/v1/reports/{id}/download     # Скачивание отчета

Особенности:
  - Кэширование результатов аналитических запросов
  - Фоновая генерация отчетов через Celery
  - Экспорт в форматы: PDF, Excel, CSV
  - Поддержка различных шаблонов отчетов
    
5.5 Notification Service (Сервис уведомлений):

Назначение: Отправка уведомлений пользователям
Технологии: Node.js + Socket.io + Nodemailer
Бизнес-требования: 2.2 (Операционные цели - уведомления)

Каналы уведомлений:
  - Email: Для важных уведомлений (оценки, дедлайны)
  - WebSocket: Для real-time уведомлений в интерфейсе
  - SMS: Для критически важных уведомлений (опционально)

Типы уведомлений:
  - New Grade: Новая оценка выставлена
  - Deadline Reminder: Напоминание о дедлайне
  - Course Update: Изменения в курсе
  - System Notification: Системные уведомления
  - Report Ready: Отчет сгенерирован

Особенности:
  - Очередь сообщений для надежной доставки
  - Шаблоны уведомлений с поддержкой мультиязычности
  - Контроль частоты уведомлений
  - Подтверждение доставки
    
6. Модель данных:
6.1 Основные сущности и их связи:

```
-- Пользователи (связь с бизнес-требованиями 3.1.1)
CREATE TABLE users (
    id UUID PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(20) NOT NULL CHECK (role IN ('admin', 'teacher', 'student')),
    full_name VARCHAR(255) NOT NULL,
    faculty_id UUID REFERENCES faculties(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- Индексы для производительности (требование 3.2.1)
    INDEX idx_users_email (email),
    INDEX idx_users_role_faculty (role, faculty_id)
);

-- Курсы (связь с бизнес-требованиями 3.1.2)
CREATE TABLE courses (
    id UUID PRIMARY KEY,
    code VARCHAR(50) UNIQUE NOT NULL,
    name_ru VARCHAR(500) NOT NULL,
    name_en VARCHAR(500) NOT NULL,  -- Мультиязычность (требование 5.1)
    credits DECIMAL(3,1) NOT NULL,
    teacher_id UUID REFERENCES users(id) NOT NULL,
    max_students INTEGER DEFAULT 30,
    is_archived BOOLEAN DEFAULT false,
    
    -- Индексы для частых запросов
    INDEX idx_courses_teacher (teacher_id),
    INDEX idx_courses_active (is_archived)
);

-- Оценки (связь с бизнес-требованиями 3.1.3)
CREATE TABLE grades (
    id UUID PRIMARY KEY,
    student_id UUID REFERENCES users(id) NOT NULL,
    course_id UUID REFERENCES courses(id) NOT NULL,
    grade_type VARCHAR(50) NOT NULL,
    value_numeric DECIMAL(5,2) CHECK (value_numeric >= 0 AND value_numeric <= 100),
    weight DECIMAL(3,2) DEFAULT 1.00,
    graded_at DATE NOT NULL,
    grader_id UUID REFERENCES users(id) NOT NULL,
    
    -- Проверки целостности
    CONSTRAINT check_grade_range CHECK (value_numeric BETWEEN 0 AND 100),
    CONSTRAINT check_weight_range CHECK (weight BETWEEN 0.01 AND 5.00),
    
    -- Индексы для аналитики
    INDEX idx_grades_student_course (student_id, course_id),
    INDEX idx_grades_date (graded_at),
    
    -- Партиционирование по датам для производительности
    PARTITION BY RANGE (graded_at)
);

-- Аудиторская таблица (требования безопасности 3.2.2)
CREATE TABLE audit_logs (
    id BIGSERIAL PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    action VARCHAR(100) NOT NULL,
    entity_type VARCHAR(50) NOT NULL,
    entity_id UUID,
    old_values JSONB,
    new_values JSONB,
    ip_address INET,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_audit_user_time (user_id, created_at)
);
```

6.2 Оптимизации для производительности:

```
-- Материализованные представления для быстрой аналитики
CREATE MATERIALIZED VIEW student_performance_cache AS
SELECT 
    student_id,
    course_id,
    COUNT(*) as total_grades,
    AVG(value_numeric) as average_grade,
    MAX(value_numeric) as max_grade,
    MIN(value_numeric) as min_grade
FROM grades
WHERE graded_at >= CURRENT_DATE - INTERVAL '180 days'
GROUP BY student_id, course_id;

-- Частичные индексы для активных данных
CREATE INDEX idx_active_courses ON courses (id) WHERE NOT is_archived;

-- Индексы покрытия для частых запросов
CREATE INDEX idx_grade_details ON grades 
(student_id, course_id, graded_at) 
INCLUDE (value_numeric, weight, grade_type);

```

8. Безопасность:
8.1 Модель безопасности:

Уровни безопасности:

  1. Сетевой уровень:
     
     - TLS 1.3 для всех соединений
     - WAF (Web Application Firewall)
     - DDoS защита через Cloudflare
     - Геофильтрация трафика

  3. Уровень приложения:
     
     - Валидация всех входных данных
     - Защита от SQL-инъекций
     - Защита от XSS и CSRF атак
     - Rate limiting по IP и пользователю

  4. Уровень данных:
     
     - Шифрование данных в rest (AES-256)
     - Хеширование паролей (bcrypt)
     - Маскирование ПДн в логах
     - Регулярные аудиты безопасности

  5. Уровень доступа:
     
     - RBAC (Role-Based Access Control)
     - JWT с коротким временем жизни
     - 2FA для административных операций
     - Сессии с инвалидацией при смене устройства
