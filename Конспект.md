## Конспект

### 1) UML (Unified Modeling Language) — это унифицированный язык графического моделирования, используемый для визуализации, спецификации, проектирования и документирования сложных систем (в основном программных, но не только).

#### 1.1 Для чего используется UML:
Визуализация — сложные идеи и архитектуру системы можно наглядно представить в виде диаграмм.
Спецификация — точное описание структуры и поведения системы (классы, интерфейсы, взаимодействия и т.д.).
Проектирование — создание моделей, которые можно анализировать и улучшать до начала реализации.
Документирование — генерация документации по архитектуре, требованиям, тестам, эксплуатации.

#### 1.2 Где применяется:
1. Разработка ПО
Объектно-ориентированный анализ и проектирование (OOAD) — диаграммы классов, объектов, последовательности.
Описание бизнес-процессов — диаграммы активностей, использования (use case).
Проектирование баз данных — диаграммы классов для моделирования схемы данных.
Тестирование — use case-диаграммы для формирования тестовых сценариев.

2. Бизнес-моделирование:
Бизнес-процессы — диаграммы активностей, BPMN (хотя BPMN — отдельный стандарт, но часто используется вместе с UML).
Организационная структура — диаграммы компонентов или развёртывания для ИТ-инфраструктуры.

3. Системная инженерия
Моделирование встроенных систем — диаграммы состояний, временные диаграммы.
Проектирование аппаратно-программных комплексов — диаграммы развёртывания.

4. Образование
Обучение проектированию ПО, паттернам проектирования, архитектуре.

#### 1.3 Основные типы диаграмм UML
Структурные (статическая структура системы):
Диаграммы классов, компонентов, развёртывания, композитной структуры, объектов, пакетов.
Поведенческие (динамика системы):
Диаграммы вариантов использования, последовательности, коммуникации, состояний, активностей, временные.

#### 1.4 Инструменты для работы с UML
Enterprise Architect, IBM Rational Rhapsody — для сложных и промышленных проектов.
Visual Paradigm, Lucidchart, Draw.io — онлайн и десктопные средства.
PlantUML — текстовое описание диаграмм с генерацией изображений.
Встроенные в IDE: IntelliJ IDEA (плагины), Visual Studio (средства моделирования).

#### 1.5 Плюсы и минусы UML
Плюсы:
Стандартизация визуального языка для команды.
Помогает выявить проблемы на ранних этапах.
Улучшает коммуникацию между заказчиками, аналитиками и разработчиками.

Минусы:
Избыточная детализация может замедлять процесс (over-engineering).
Модели могут устаревать, если их не поддерживать синхронно с кодом.
Не все диаграммы одинаково полезны на практике (используют чаще всего 5–6 основных типов).

#### 1.6 Современное применение
Сейчас UML реже используется в полном объёме, но отдельные диаграммы (например, use case, последовательности, классов) широко применяются в архитектурном проектировании, документировании API и микросервисной архитектуре.
В гибких методологиях (Agile) UML используется точечно, по мере необходимости, без создания подробной документации на начальном этапе.
