# Синтаксис — это "грамматика" языка. Он отвечает на вопрос "КАК это должно быть написано?", но не объясняет "ЧТО это делает?" (это уже семантика).
## Ключевые аспекты синтаксиса:

- Правила записи конструкций: Порядок ключевых слов, операторов, скобок, кавычек.
Пример: В Python блок кода выделяется отступами, а в C++ или Java — фигурными скобками {}.

- Правила именования: Какие символы можно использовать в именах переменных, чувствительность к регистру.
Пример: В Bash имена переменных не могут начинаться с цифры, в PowerShell они начинаются с $.

- Правила оформления: Где ставить точки с запятой, запятые, двоеточия.
Пример: В JavaScript точка с запятой ; часто ставится в конце инструкции, в Python она не обязательна.

- Структура выражений и команд: Как комбинировать элементы для формирования осмысленных инструкций.
Пример: В Bash команда if должна заканчиваться словом fi, а в PowerShell используется if { ... }.

### Синтаксис vs. Семантика
- Это ключевое различие:
- Синтаксическая ошибка: Код нарушает правила написания. Программа не сможет даже начать выполнение (интерпретатор/компилятор выдаст ошибку).
Пример: prinnt("Hello") (неправильное имя функции), отсутствие закрывающей кавычки ", пропущенная }.

- Семантическая ошибка: Код синтаксически правилен, но делает не то, что задумано. Программа выполнится, но результат будет неверным.
Пример: if (x = 5) вместо if (x == 5) (в некоторых языках это присваивание, а не сравнение), бесконечный цикл по логической ошибке.

#### Почему синтаксис важен?
- Однозначность: Позволяет компилятору/интерпретатору (или человеку) однозначно разобрать текст инструкций.
- Стандартизация: Обеспечивает единый стиль для всего сообщества разработчиков.
- Автоматизация: Инструменты (подсветка синтаксиса, автодополнение, линтеры) полагаются на строгие синтаксические правила.
- Эффективность: Многие синтаксические решения влияют на читаемость и удобство написания кода.
- Итог: Синтаксис — это фундаментальный свод правил правописания для любого формального языка, будь то язык программирования, разметки или командной оболочки. Без него не было бы возможности автоматической обработки инструкций компьютером.

##### Примеры синтаксиса в разных контекстах:
###### 1. Языки программирования (Python, Java, C#)
- Синтаксис определяет структуру кода.
```
# Python: Синтаксис включает двоеточие после условия и отступы.
if x > 10:
    print("Больше десяти")
```
```
// Java: Синтаксис требует круглые скобки для условия, фигурные для тела, точку с запятой.
if (x > 10) {
    System.out.println("Больше десяти");
}
```
###### 2. Языки разметки (Markdown, HTML, XML)
- Синтаксис определяет, как текст оформляется в документе.
```
# Markdown: Синтаксис заголовка — решетка в начале строки.
## Это подзаголовок (две решетки)
*Это курсив* (звездочки вокруг текста)
```
```
<!-- HTML: Синтаксис использует теги в угловых скобках. -->
<h1>Это заголовок</h1>
<em>Это курсив</em>
```
###### 3. Скриптовые языки (Bash, PowerShell)
- Синтаксис определяет формат команд и их аргументов.
```
# Bash: Синтаксис использует $ для переменных, пробелы критически важны.
if [ "$name" == "Вася" ]; then
    echo "Привет, Вася!"
fi
```
```
# PowerShell: Синтаксис использует $ для переменных, фигурные скобки и cmdlets.
if ($name -eq "Вася") {
    Write-Host "Привет, Вася!"
}
# Параметры начинаются с дефиса: Get-Process -Name "chrome"
```
###### 4. Языки описания (UML, YAML, JSON)
Синтаксис определяет, как описывать данные или диаграммы.
```
# YAML: Синтаксис основан на отступах и двоеточиях.
server:
  host: "localhost"
  port: 8080
```
```
// JSON: Синтаксис требует фигурные/квадратные скобки, кавычки для ключей.
{
  "server": {
    "host": "localhost",
    "port": 8080
  }
}
```
```
' UML (на примере PlantUML): Синтаксис использует ключевые слова для типов стрелок.
@startuml
Актер -> Система: Запрос
Система --> Актер: Ответ
@enduml
```
